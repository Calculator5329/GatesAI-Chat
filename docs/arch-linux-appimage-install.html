Install GatesAI Chat on Arch Linux

Use this guide to install the GatesAI Chat AppImage on Arch Linux and confirm that the bundled Linux bridge starts correctly.

1. Install Runtime Packages

Install the AppImage and WebKit runtime packages from the Arch repositories:

sudo pacman -Syu
sudo pacman -S fuse2 webkit2gtk-4.1

fuse2 lets AppImage files mount and run. webkit2gtk-4.1 provides the WebView runtime used by Tauri on Linux.

2. Copy The AppImage

Create a permanent folder for user-installed AppImages:

mkdir -p "$HOME/Applications/GatesAI Chat"

If the AppImage is in ~/Downloads, copy it into that folder and make it executable:

cp "$HOME/Downloads/GatesAI Chat_3.7.0_amd64.AppImage" "$HOME/Applications/GatesAI Chat/"
chmod +x "$HOME/Applications/GatesAI Chat/GatesAI Chat_3.7.0_amd64.AppImage"

3. Launch

Run the AppImage directly:

"$HOME/Applications/GatesAI Chat/GatesAI Chat_3.7.0_amd64.AppImage"
After launch, open Menu -> Workspace. The status should become Bridge online.

4. Add A Desktop Launcher

Create an applications-menu entry:

mkdir -p "$HOME/.local/share/applications"
cat > "$HOME/.local/share/applications/gatesai-chat.desktop" <<'EOF'
[Desktop Entry]
Type=Application
Name=GatesAI Chat
Comment=Local-first AI chat with workspace tools
Exec=/home/YOUR_USER/Applications/GatesAI Chat/GatesAI Chat_3.7.0_amd64.AppImage
Terminal=false
Categories=Utility;Development;
EOF

Patch the launcher path to your Linux home directory:

sed -i "s|/home/YOUR_USER|$HOME|" "$HOME/.local/share/applications/gatesai-chat.desktop"
update-desktop-database "$HOME/.local/share/applications" 2>/dev/null || true

Log out and back in if the launcher does not appear right away.

5. Verify The Bridge

If the app says the bridge is offline, check whether another process is already using the bridge port:

ss -ltnp | grep ':7331' || true

You can also run the AppImage from a terminal to watch bridge startup logs:

"$HOME/Applications/GatesAI Chat/GatesAI Chat_3.7.0_amd64.AppImage"

6. Troubleshooting

If the AppImage does not start and mentions FUSE, reinstall fuse2:

sudo pacman -S fuse2

If the app opens to a blank window or exits with WebKit-related errors, reinstall the WebKit runtime:

sudo pacman -S webkit2gtk-4.1

If the app launches but workspace tools are offline, make sure this AppImage was built with the real Linux bridge sidecar, not the packaging-only stub.

7. Inspect The AppImage

To confirm the bridge binary is inside the AppImage:

mkdir -p /tmp/gatesai-appimage-check
cp "$HOME/Applications/GatesAI Chat/GatesAI Chat_3.7.0_amd64.AppImage" /tmp/gatesai-appimage-check/
cd /tmp/gatesai-appimage-check
chmod +x ./*.AppImage
./*.AppImage --appimage-extract >/dev/null
find squashfs-root -name 'gatesai-bridge*' -exec file {} \;

The bridge entry should report an ELF 64-bit executable.

Package names verified against the Arch Linux package database for fuse2 and webkit2gtk-4.1.
