# Glossary

## Artifact

A file generated by the assistant for the user to view or use, such as an HTML
preview, document, image, or workspace output.

## Bridge

The local desktop companion process that gives the app controlled access to the
computer. It handles workspace files, shell commands, git, and local runtime
connections that the browser cannot safely perform.

## Capability

A named thing the app can do in the current runtime, such as "use OpenRouter
chat", "read workspace files", "generate with ComfyUI", or "run terminal
commands." Capabilities should decide what the UI and model can access.

## ChatTurnEngine

Planned extraction from `ChatStore`. It should coordinate one assistant turn:
prompt assembly, streaming, tool calls, tool results, continuation, cancellation,
and final message state.

## ComfyUI

The supported local image generation runtime. GatesAI should treat it as the
main local-image backend rather than trying to support every image tool.

## Core

Shared project language: types, provider contracts, model metadata, tool
metadata, capability definitions, and runtime concepts.

## Desktop

The Tauri app. This is the full product because it can use the bridge and local
workspace.

## Image job

A queued image generation task. It should render as a card in chat, update its
own progress, and complete in place.

## LEGO refactor

The direction of making the app easier to understand by turning workflows into
small blocks with clear jobs: capabilities, providers, tools, stores, services,
artifacts, image jobs, and chat turn orchestration.

## OpenRouter

The supported cloud model provider. It gives GatesAI pay-as-you-go access to
many model families through one API key.

## Ollama

The supported local LLM provider. It lets users run local models on their own
machine.

## Provider

An adapter that talks to a model runtime or API. GatesAI currently focuses on
OpenRouter and Ollama for chat, plus OpenRouter images and ComfyUI for images.

## Service

A focused work function or module. Services talk to external boundaries, format
results, persist data, or perform reusable logic. They should not secretly own
UI state.

## Store

A MobX state owner. Stores remember what is happening in the app and expose
actions the UI can call.

## Tool

A callable action the model can request, such as reading a file, searching the
web, running a command, generating an image, or writing an artifact.

## Web Lite

The browser demo. It should show the real GatesAI interface and let users try
OpenRouter chat, but it should not pretend to have desktop bridge powers.

## Workspace

The local folder GatesAI is allowed to use for files, artifacts, notes, and
future source-workspace operations.
